GOV.UK One Login

This test system is for developers to create and test integrations with GOV.UK One Login.

It provides test endpoints used during the authentication and identity verification process. These return example responses that can be used to validate your integration and perform testing.

Integrating

  • Redirect to GOV.UK One Login authorization server
  • Exchange authorization code for an id token and access token
  • Verify id token
  • Use access token to retrieve user information
  • Validate core identity claim
  • Check failure scenarios

OpenId Connect endpoints

  • /.well-known/openid-configuration

    You can use this URL to automatically load OIDC client configuration.

  • /authorize

    Start the authentication and identity proving process by redirecting a user's browser here with the right query parameters.

  • /token

    Token exchange endpoint. Called by server-side code to swap the auth code for an access token.

  • /userinfo

    Protected by the access_token. Called by server-side code to access the result of the identity verification.

Configuration

  • Keys

    View the public key used to sign the core identity claim returned as a result of identity verification

  • Client configuration

    View the details of the configured OpenID Connect clients.
